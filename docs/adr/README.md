# Architecture Decision Records — index

One row per ADR: number, title, status, and the one-line decision it records. ADRs are a
**historical record** — read the current architecture in `docs/architecture/00-overview.md` and
the cross-service contracts in `docs/contracts/`; come here to understand *why* something is the
way it is, or to see what an earlier design tried before it was superseded.

| # | Title | Status | What it decided |
|---|---|---|---|
| [0001](0001-remote-mcp-developer-setup.md) | Remote MCP server as the developer setup (Option A) | Accepted | Developers connect to a remote MCP server; the only local footprint is an AI client, agent files, MCP config, and a repo checkout — no local KB infrastructure. |
| [0002](0002-postgres-knowledge-registry.md) | Postgres as the Knowledge Registry and source of truth | Accepted | Azure PostgreSQL Flexible Server is the single source of truth; Azure AI Search is a derived, rebuildable projection only. |
| [0003](0003-graph-in-postgres-not-graph-db.md) | Graph model in Postgres; no graph database in V1 | Accepted | Store nodes/edges in Postgres (`knowledge_edge`); expose graph behavior only through MCP graph tools, no dedicated graph DB in V1. |
| [0004](0004-nightly-incremental-build.md) | Nightly incremental batch build (not streaming) | Accepted | Build the KB nightly via scheduled CI, incrementally by content hash, with every model call cache-gated; activate only after validation. |
| [0005](0005-mcp-context-broker-evidence-pack.md) | MCP Context Broker + shared Evidence Pack | Accepted | The MCP server is a Context Broker: run-scoped Evidence Pack, server-enforced budgets, evidence cards before raw text, retrieval ledger. |
| [0006](0006-stack.md) | Implementation stack (Python / uv / fastmcp / Alembic) | Accepted | Concrete V1 stack: Python 3.12 + uv, async fastmcp, SQLAlchemy 2.x + asyncpg + Alembic, `SearchClient`/`ModelClient` interfaces, ruff/pyright/pytest. |
| [0007](0007-v1-exclusions.md) | V1 resource exclusions and add-when triggers | Accepted | Excludes Azure Functions, Event Grid/Service Bus, Redis, APIM, Blob Storage, a graph DB, prod SQLite, and streaming from V1; re-entry needs a new ADR. |
| [0008](0008-self-contained-services.md) | Two self-contained deployable services | Accepted | Restructures the build/runtime planes into fully self-contained `uv` projects under `services/` with no shared runtime packages — contracts are markdown docs. |
| [0009](0009-portable-agent-framework.md) | Portable agent framework: host-native renderings of the agent manifests | Accepted | `agents/*.md` is the canonical manifest set; `.copilot/` and `.opencode/` are hand-authored, parity-pinned renderings for other hosts. |
| [0010](0010-unified-build-and-extractor-split.md) | Unified `build` entry point with an internal extractor split | Accepted | One product-facing `build` command ingests code/docs/cards into one graph, with an internal deterministic-extractor vs. LLM-prose split. |
| [0011](0011-end-to-end-trust-contract.md) | End-to-end trust contract: trust buckets, claim/evidence ledger, verifier, signed receipts | Accepted | Adds a trust/evaluation contract spanning source → graph → retrieval → answer: trust buckets, a claim ledger, a verifier, and signed receipts. |
| [0012](0012-graphify-extraction-backend.md) | Adopt Graphify as the multi-language code-extraction backend | Accepted | Adopts the Graphify library (tree-sitter AST, ~20 languages) as the deterministic code extractor instead of hand-rolling a Python-only one; amends ADR-0010. |
| [0013](0013-kb-version-membership-and-identity-over-time.md) | `kb_version` membership model + identity-over-time | Accepted | Interval-membership model (`valid_from_seq` / `invalidated_at_seq`) so incremental nightly builds and MCP serving stay consistent with ADR-0004. |
| [0014](0014-observability-and-cost-dashboard.md) | Observability + cost-analysis dashboard over the ledger and eval metrics | Accepted | Operator-only Phase 1 dashboard: four aggregate views over `retrieval_event`/`kb_build_run`, including the `kb_search` zero/thin-result KB-gap proxy. |
| [0015](0015-production-source-connectors.md) | Production source connectors (GitHub now; ADO Wiki + Work Items next) | Accepted | Real `FetchBackend` implementations for GitHub, ADO Wiki, and ADO Work Items via PAT auth, extending ADR-0004/0010. |
| [0016](0016-local-dev-auth.md) | Local developer auth for exercising MCP tools | Accepted | Opt-in, OFF-by-default local-dev auth verifier (`MCP_LOCAL_DEV_AUTH`) with hard boot-time guardrails; production auth is byte-for-byte unchanged. |
| [0017](0017-local-search-projection-parity.md) | Persistent local search projection (local ↔ Azure parity) | Accepted | Local-dev implementation of the `SearchClient` interface with the same index-persistence semantics as Azure AI Search. |
| [0018](0018-code-is-graphify-only.md) | Code is graphify-only: deterministic spans + metadata, never the LLM | Accepted | Code extraction is entirely deterministic (Graphify); the LLM is reserved for prose sources and the cross-domain relationship judge. |
| [0019](0019-ollama-embeddings-enable-semantic-linker.md) | Ollama embeddings provider + enable the semantic linker | Accepted | Adds a local Ollama embeddings provider and enables the semantic linker, bridging the previously weakly-joined code/prose graph islands. |
| [0020](0020-structural-code-edges.md) | Structural code edges: `defined_in` (symbol→file) and `imports` (file→file) | Accepted | Adds deterministic structural code edges so an agent can navigate from a symbol to its file and across file dependencies. |
| [0021](0021-human-approval-delegation-gate.md) | Human-approval gate at every agent delegation | Accepted | Every orchestrator → subagent handoff passes through a human-approval checkpoint, logged to the retrieval ledger. |
| [0022](0022-agent-gateway-code-owned-routing.md) | Code-owned intent routing; "broker-governed" vs "broker-assisted" | Accepted | Intent routing and policy enforcement are code-owned, not implied by the markdown orchestrator being a universal entry point. |
| [0023](0023-docs-are-graphify-llm-extracted.md) | Documents are Graphify-LLM extracted; retire the hand-rolled wikify pipeline | Accepted | Extends Graphify from code-only to prose/document extraction, retiring the hand-rolled `wikify` chunker + LLM pipeline. |
| [0024](0024-two-mode-tool-governance.md) | Two-mode tool governance: the model gets no shell; the runner owns execution | Superseded by ADR-0025 + ADR-0026 | Tried a no-shell/RPC-only tool surface for agents; retained for the historical reasoning trail — it over-gated the model. |
| [0025](0025-kb-first-file-fallback.md) | KB-first with file-fallback: the knowledge base is a budgeted tool, not a gate | Accepted (supersedes ADR-0024) | Reverses the mandatory broker-only flow: agents get native file/grep/search tools back plus a budgeted `kb_search`; the broker pack path becomes optional. |
| [0026](0026-code-skeleton-compression.md) | Code-skeleton compression: shrink what the agent reads, never gate it | Accepted | Compresses code reads to skeletons (signatures kept, bodies elided) with the exact body one `read_full` away, instead of restricting access. |
| [0027](0027-crash-durable-model-output-cache.md) | Crash-durable model-output cache: never re-pay for tokens a crashed build already spent | Accepted and Implemented (PR-35); superseded in part by ADR-0029 | Durable content-hash-keyed cache for docify/embedding model outputs so a crashed build never re-pays for already-spent tokens. |
| [0028](0028-graph-centrality-ranked-retrieval.md) | Graph-centrality ranked retrieval: rank by the graph we already store | Accepted and Implemented (PR-36) | Adds a normalized PageRank `centrality_score` on `knowledge_artifact` and folds it into the broker's rank key as a transparent, backward-safe prior. |
| [0029](0029-incremental-per-source-commit.md) | Persist knowledge per source: incremental commit, atomic activation | Accepted (supersedes ADR-0027's atomic-write stance) | Commits each source's artifacts/edges as it finishes, instead of one end-of-build transaction, so one source's failure no longer aborts the whole build; activation stays atomic. |
| [0030](0030-twelve-role-roster-and-langgraph-backend.md) | Twelve-role agent roster, with LangGraph orchestrating MCP tool responses and the review panel | Accepted | Expands the roster to twelve roles; LangGraph orchestrates the `get_task_context` backend and the review panel; evaluates and rejects Anthropic Managed Agents. |
| [0031](0031-dev-gated-review-publication.md) | Dev-gated review publication: the panel drafts, the developer publishes | Accepted (amends ADR-0030 §3) | The review panel drafts on-demand from a developer's own session and never auto-posts; the developer always reads, revises, and publishes. |
| [0032](0032-postgres-first-tracing-behind-a-port.md) | Traces live in Postgres behind a TraceSink port; no hosted tracing SaaS | Accepted (amends ADR-0030 §4) | Withdraws the LangSmith commitment before it ever activates; per-step traces go to Postgres (`trace_span`) behind a `TraceSink` port instead. |
