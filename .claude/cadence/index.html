    You're set up.

    The timeline lives here. Every prompt you send, every tool call Claude runs, every sub-agent dispatched, every PR opened — it all lands in this view, automatically, as it happens.

    01 · What you'll see

    One node per event

    Your prompts as cards, Claude's responses as cards, tool calls bucketed inside each turn (read · edit · bash · web · subagent · failures), decisions called out with apricot tint.

    02 · Parallel sub-agents

    Fork → N lanes → merge

    When Claude dispatches sub-agents in parallel, they render as a multi-lane block with fan-out / fan-in SVG curves and a contribution card showing what each lane shipped.

    03 · Live

    5-second poll

    This page re-fetches the data file every 5 seconds. No SSE, no WebSocket — just plain HTTP polling against a local file. Open it once and let it run.

    04 · Local

    Nothing leaves your machine

    Zero LLM cost, zero telemetry, zero hosted services. The plugin watches what Claude Code is already doing and writes a JSON file your browser polls.
